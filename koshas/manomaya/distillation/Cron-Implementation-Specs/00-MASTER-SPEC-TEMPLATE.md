### **6. Failure Handling (Vikara Restoration Protocols)**

**Common Failure Modes:**

**1. Schedule Failure (Missed Run)**
- **Vikara:** Moha (delusion, system confusion)
- **Detection:** `state.lastRunAtMs` > `schedule.nextRunAtMs + 600000` (10 min late)
- **Restoration:** Execute immediately, log to `_System/cron/logs/[job-name]-recovery.log`, alert to Telegram

**2. Delivery Failure (Message Not Sent)**
- **Vikara:** Ahamkara (ego inflation, "I sent it" when it didn't arrive)
- **Detection:** `message` tool returns error, or NO_REPLY when delivery expected
- **Restoration:** Retry with exponential backoff (1min, 5min, 15min), fallback to alternate channel (Telegram ‚Üí WhatsApp)

**3. Audio Generation Failure (sag quota exceeded)**
- **Vikara:** Lobha (greed, over-consumption of API quota)
- **Detection:** `sag` returns 429 or quota error
- **Restoration:** Fallback to macOS `say -v Samantha`, note in message text: "(ElevenLabs quota exceeded ‚Üí macOS fallback)"

**4. Attachment Missing (opus stub not found)**
- **Vikara:** Moha (delusion, incorrect path)
- **Detection:** File read fails on expected stub path
- **Restoration:** Fall back to `octave-8.opus`, include `MISSING_STUB:<expected>` in text, alert to Telegram

**5. Kosha Misalignment (Wrong layer for task)**
- **Vikara:** Mada (intoxication, pride in complexity)
- **Detection:** Job operates at wrong Kosha (e.g., Vijnanamaya task forced into Annamaya execution)
- **Restoration:** Abort execution, log error, dispatch to correct Kosha-appropriate agent

**6. Guna Drift (Sattva ‚Üí Rajas or Tamas)**
- **Vikara:** [Specific affliction based on drift direction]
  - Sattva ‚Üí Rajas: Kama (desire for speed), Krodha (frustration with pace)
  - Sattva ‚Üí Tamas: Moha (confusion), Mada (apathy, "who cares")
- **Detection:** Execution time exceeds expected duration by >50%, or user reports "feels forced"
- **Restoration:** Inject Aletheios (if Rajas) or Pichet (if Tamas), pause 60s, retry with balanced Guna

**Example:**
> **Failure Mode:** Audio generation fails (quota exceeded)
> **Vikara:** Lobha (over-consumption)
> **Restoration:** Fall back to macOS `say`, note in text, reduce daily voice generation to conserve quota
