### **7. Failure Handling (Vikara Restoration Protocols)**

**Common Failure Modes:**

**1. Image Generation Failure (FAL.ai quota/error)**
- **Vikara:** Lobha (over-consumption of API quota)

- **Restoration:** Execute fallback cascade:
  1. Retry with `fal-flux` (standard quality)
  2. Retry with `fal-flux-schnell` (rapid)
  3. Proceed text-only, note in content: "(Image generation quota exceeded, text-only transmission)"
- **Log:** Append to `_System/cron/logs/twc-content-generator-daily.log`

**2. Voice Generation Failure (ElevenLabs quota)**
- **Vikara:** Lobha (over-consumption of voice quota)
- **Detection:** `sag` returns 429 or quota error
- **Restoration:** Fall back to macOS `say -v Samantha`, note in daily log: "(ElevenLabs quota exceeded, macOS voice fallback)"
- **Alternative:** Skip voice generation, proceed text-only

**3. Field Architect Framework Missing**
- **Vikara:** Moha (confusion, file path error)
- **Detection:** Cannot read Field Architect `.md` file from `/Volumes/madara/2026/twc-vault/_System/Templates/Content-Processing/Field-Architects/`
- **Restoration:** Default to **Field Resonator** (most versatile), alert in daily log

**4. PARA Rotation State Corrupted**
- **Vikara:** Moha (delusion in state tracking)
- **Detection:** `content-rotation-state.json` missing or malformed
- **Restoration:** Reinitialize state file:
```json
{
  "lastPlatform": "LinkedIn",
  "lastArchitect": "Synaptic Sage",
  "lastRunDate": "2026-02-05",
  "cycleDay": 0,
  "totalRuns": 0
}
```
- Continue with Day 1 (X/Twitter, Field Resonator)

**5. Vikara Detection in Generated Content**
- **Vikara:** Kama (virality chasing), Ahamkara (ego inflation), Dambha (hypocrisy)
- **Detection:** Manual review by human finds content misaligned with authentic transmission
- **Restoration:** Add `REJECTED-` prefix to filename, log reason in daily log, human revises or skips publication
- **Learning:** Adjust Field Architect prompt templates to reduce Vikara likelihood

**6. Schedule Failure (Missed Run)**
- **Vikara:** Moha (system confusion)
- **Detection:** `state.lastRunAtMs` > `schedule.nextRunAtMs + 600000` (10 min late)
- **Restoration:** Execute immediately, mark as "recovery run" in daily log
